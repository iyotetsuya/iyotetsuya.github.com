DroidKaigi 2019 現場

活動連結

日本 Android 開發者的每年一度盛會！
今年的參加人數規模己接近千人左右，今年開始還有一部份的 Session 提供英日的雙向通譯。

參加費用的部份

  • 早鳥票： ¥10000
  • 一般入場： ¥15000

建議要注意開賣的時間，早鳥票很快就會賣完，一般票也是半天就會完售的程度。
這次沒搶到早鳥，但還好公司有提供參加費的補助。
同事表示看來 Android 越來越有要超過iOS的跡象。
但真的不幸沒有買到的話也沒有關係。
所有的 Session 都會有工作人員架器材錄影，之後可以在 Youtube 上觀看。

參加的 Session 與心得

  1. マルチモジュールなプロジェクトでテストはどう変わる？

    有提到 PIT 這個工具可以做 Mutation Testing 。

  2. マルチモジュールプロジェクトでの Dagger2 を用いた Dependency Injection

    利用 DI 將圖像讀取、http client 等第三方 lib 分離來抽象化，方便之後需要替換的情況。
    使用 Dagger-Android 減少 Fragment 跟 Activity 在 Inject 時需要注入細節的情況。

  3. Espresso のテストを Android の最新トレンドに対応させよう

    蠻建議實際練習一次裡面 PageObject 的實作過程，在做 UI 測試可以省下不少時間。
    投影片和範例
    但前提條件是需要先將其它背景處理的部份先 mock 好

  4. ぼくのかんがえた最強の Usecase の作り方~あるいはビジネスロジックとはなにかという1つの回答~

    如果在切程式碼的時候，有什麼才算是業務邏輯？這樣疑惑的人可以聽看看。

  5. The good and bad of modern app architecture

    在導入技術時，應該也要同時考慮到未來新進人員 Onboarding 時的成本。
    而不僅僅是減少目前的開發成本而己。

  6. 中規模以上のアプリ開発における CI レシピとリリースフロー戦略

    投影片
    推薦給沒用過 Jenkins 等 CI tool 或是剛開始學習 git-flow 的人。
    相對來說比較偏向入門的 Session。

  7. Guide to app architecture を踏まえた既存アプリの設計改良

    在有技術債時，重點應放在要如何增加維護性來決定改善方向。
    投影片和2018年的資料
    但多的是想要全部打掉重練，結果把資源全部投入無底坑的例子。

  8. 実践 Lottie

    投影片
    基本上對Designer來說負擔比較大，需要對AE有一定的熟悉程度。
    （從0開始做一個畫面大概會花到4天，習慣的話1天左右）
    需注意有一些濾鏡、效果是沒有支援的。（有平台差異）
    也不太適合需要與使用者互動的動畫，像是依照手勢移動的動畫。
    另外把動畫的 json 檔放在 Server 端下載的話還可以做像是 A/B test 的事情。

  9. ハマった時に役立つ通信系デバッグの話

    投影片
    Android Studio 中的 Network Profiler （一直以來都是用Postman在確認
    Proxy 可以用來限制網路通信量，但基本上還是用模擬器比較方便。

  10. Androidにおけるパフォーマンスチューニング実践

    https://developers.google.com/protocol-buffers/
    https://github.com/google/battery-historian

  11. デザイナーとエンジニアの距離をより近づける Lottie 利用術

    https://lottiefiles.com/
    https://github.com/willowtreeapps/Hyperion-Android

通譯

不過收音的部份是用同一台機器的關係，會變成二種語言同時播放，
如果能聽懂大部份的話，個人比較建議不要依靠通譯會比較好。

另外大約在一月左右會在 Github 上開 Android 的 Repo，
如果有想要做為貢獻者被放在大會開場的投影片上可以去發 PR 。

DeNA TechCon 2019

活動連結

公司的同事裡有認識的人在DeNA上班，就經用社員招待的方式參加。
不用參加費，但社員招待不受到一般席的數量限制。
順便看DeNA家的產品，也可以當做轉職時的參考。
而且它辦在平日，所以能當薪水小偷

參加的Session與心得

錄影的部份，官方會日後上傳的樣子，有了之後會再更新連結。

  1. ヘルスケアサービス開発の裏側 〜品質と開発効率の両立〜

    有一點讓我覺得標題跟內容不是那麼的吻合。
    前半段：割愛，沒留下什麼特別印象。
    後半段：

    • 自己的狗糧自己吃
      不賣連自己都不用的東西。雖然之前就聽過了，但這個概念還是很值得再次思考。
    • Swagger
      可以解決在少人數圑隊常有的文件老舊問題。
      連實作的時間都沒有了，哪有時間寫文件！
      但目前的工作上的有其它同事嘗試過，如果要從既存API遷移過去的話，需要不小的成本，目前擱置中。

  2. スマホゲームのチート手法とその対策

    如果有用過GameMaster跟封包截取的軟體的話，可以不需要特別看這個Session，因為基本原理差不多。
    大致上來說有分為對記憶體、封包、程式碼還原、暫存文件等作弊方式。
    其中有針對封包截取的部份開發自製工具（提供開發工具做加解密等），預計會在最近公開，有興趣的話可以注意一下他們的Github。

  3. DeNAの品質を支えるQAの取り組み 〜標準化から実践まで〜

    實際上測試計畫投入的內容，雖然說是有興趣去聽的，但這邊的測試比較偏向本業為QA的內容，
    離我會的有點遠反而沒什麼可以說的。

  4. ある SWET エンジニアの開発プロセス改善最前線

    裡面剛好有提到UI檢查的部份，就順帶一提。
    Android在這方面可以在自動化測試時留下螢幕截圖減少手動抓圖的時間。（但還是要人工比對就是了）
    比如說像是FB的screenshot-tests-for-android

其實還有參加二個Session跟沒去參加的交流會，覺得不是很重要就不放了。
這個活動每年都差不多在二月上旬左右舉行，有興趣的人歡迎明年也去參加看看～

對於改善生活的方法

每到新的一年就會開始無意義的計畫大概是人類的劣根性
還記得2018年的目標是開始經營youtube並達到10萬人訂閱，結果一年過了只有13個訂閱數
所以人需要先看清楚自己做的到的事
然後吐嘈一下別人是新年在做，你是一月都要結束了才在寫

一開始先檢查目前的生活，整理狀況跟思考要怎麼改善，
因為工程師的職業病，先做第一次的KPT（keep, problem, try）。

Keep

想保持的部份：

  • 薪水
    • 不算到很好，但至少不用在買想要的東西時斤斤計較
    • 最近剛買了Switch的伊布同捆，之後再寫一篇怎麼樣搶到的
  • 一年國內旅行2~3次（二天以上）
    • 2017: 日光（7月3天2夜）、北海道（12月4天3夜）
    • 2018: 大阪（4月3天2夜）、箱根（9月3天2夜）、三宅島（12月3天2夜）
    • 之後出去玩應該會來寫寫遊記
  • 不用加班
    • 大致上都是7:30~4:30，一個星期有一天可以在家工作

Problem

想改善的事情：

  • 增加旅遊的次數
    • 都難得住在日本，一直待在東京總覺得很可惜。
  • 工作最近有點變動，己經不滿足當初的目標
    • 想要可以用日文做為主要語言、做出來的產品是自己也會想要用的東西
  • 日文沒什麼長足的進步

Try

準備嘗試去做的事：

  • 每個月做一次KPT
    • 在行事曆上加上提醒日
  • 計畫1~3月的過夜旅遊
  • 每周寫一篇廢文文章

畢竟沒有持續進行自我檢視就很難說得上是什麼改善 ODO

Android Wantedly日本面試之旅(番外篇)

想不到拖了一下稿就工作到滿二年了XD
這篇主要寫日本生活部份的建議與感想，
皆以東京做為前提。

租房子要租哪裡？

工作是找新創公司話，推薦從渋谷找會是一個不錯的選擇，
很多新創公司都在這一區或是附近而己。

如果覺得貴的話，可以找同一條線上不用轉車，
有電車始發的車站會來的比較便宜跟方便。
不然每天在滿員電車的洗禮下很快就會厭世到想要回台灣…

至於其它地區的話，也建議先去用「混雑率」當關鍵字去避開前幾名的路線，
女性的話最好再看一下有沒有痴漢風評的部份。
畢竟日本多的是每天花一、二個小時通勤的人…
（個人無法，今年四月才搬到可以騎腳踏車15分鐘通勤的距離）

一個月開銷大約是？

主要取決在房租跟飲食習慣，
平均來說1K的房型大約在7萬元上下，
外食的話商業午餐約是1000元，
但整體的個人經驗來說，日元 * 0.3 / 2 大概就是在台北花台幣的感覺。
當然也有這樣算下來很便宜的部份……像是牛奶。

要怎麼交日本朋友？

工程師的話有 https://connpass.com/ 可以找同領域的，
都會有定期舉辦的讀書會或是活動可以參加

或者是中英日文的語言交換活動。
其中日←→英的應該會比較好，
目前遇到會真的努力學中文+參加活動的日本人真的很少
真的能夠用中文溝通的，通常都是混血兒或是以前在台灣或中國生活過。
而如果你是20~35歲 + 單身的話在這部份應該會再容易一點（？）
也會人在這些活動上找交往對象（當然傳銷的也有就是）

我自己的狀況是只有同事比較熟一點……
從活動認識的還沒有遇到可以直接說常體的朋友，
目前正朝桌遊圈跟推特上努力中。

在日本生活的雜感

  • 前一陣子回台灣過馬路時差點被車撞
  • 日本很多事情不是因為守規矩，而是怕麻煩且不想承擔可能的額外責任。
  • 只會英文也可以活下去，但會日文可以活的比較輕鬆。
  • 有點閒錢又有點時間的話 生活品質比起台北好很多
  • 季節變化很棒，空氣很好，但開始擔心明年春天會不會得花粉症
  • 日文靠自學可以拿到N1，但文法跟口說就是另一回事
  • 剛開始信用卡難辦到爆，先把日本駕照換起來比較好
  • 會講一點中文或大學時中文是第二外國語的日本人不少
  • 開始猶豫是不是不會回台灣
  • 討厭煙味的人會很痛苦，有些場合避不掉。

其它

如果還有什麼其它想知道的事，
也歡迎留言在此篇文章之中，
希望以上能對其它想前往日本發展的人提供一些幫助。

個人在職中還是陸續有被其它公司主動約面試，
我想招收外國人的需求是一直都在的，
如果有想換個環境的話不妨也來日本挑戰看看吧，
以上。

（差點忘了怎麼把Hexo裝回來）
（終於可以完結這個系列不用再拖稿了）

Android Wantedly日本面試之旅(4)

請參閱之前的內容
Android Wantedly日本面試之旅(1)
Android Wantedly日本面試之旅(2)
Android Wantedly日本面試之旅(3)

距離上次發的心得文己經半年了，
最近被催稿想說是該來還債了XD

但這篇與其說是面試的心得分享，
不如說是選了目前上班公司的原因與實際上班的心得，
對於了解實際面試的部份，
參考性不大。

ジョイズ株式会社

公司主動連絡，
服務是用App做AI的英文會話練習，
用AI就不用考慮到會話練習的時間點限制，
還有請老師反覆練習同一段落會有的尷尬感，
算是蠻有趣的切入點。

公司的社員數約10人，
共經歷二次面試。
第一次主要是針對個人背景，
很定番的為什麼要來日本等等。
做過什麼類型的App，
然後了解一下對於這份工作的意向如何。

比較特別的是因為是語言學習的服務，
所以相對其它面試經驗來說，
自己也在學日文所以比較有共鳴感，
留下的印象也比較深刻。

老闆本身也是去過英國留學回來的工程師，
服務很多部份也是自己下去實作，
聊完覺得雙方似乎都蠻符合需求的，
就敲定一個星期後做技術面試。

技術面試本身就是很一般的白板面試，
會詢問寫法的理由跟覺得哪部份可以改善的，
以前沒實際遇過這類型的面試方式，
建議如果有時間還是多練習才不會失常。
自己當天是蠻緊張的，
實際上沒解完問題，
還被提出了幾個可以改善的地方。

白版面試完有稍微提了一個邏輯思考的問題，
但我覺得不是很有參考性，
頂多算是看反應能力而己。

回答完問題之後當下就直接拿到內定通知書。
（我猜是先印好，沒過就直接說之後再連絡了）

結果：當天通知採用，年薪450萬。

以接受的理由來說，
薪資就只是剛好到自己的底線，
如果純粹考量薪資的話，
評估以自己的資歷來說，
要找到年薪600萬的工作也不是不可能的事，
如果再加上日文工作上沒什麼問題的話，
我認為不難達成，
這部份提供給有相同目標的人參考。

但以自己的面試目標來說，
這間公司己經達到了，
還額外多了可以學習英文的環境，
服務本身也是自己會想用的內容。
在這樣考量下，
想再找到其它類似的工作機會應該不太容易，
就決定不再接受新的面試邀約，
敲定上班時間開始新職場生活。

這中間大約投了一百多間公司，
面試約三十間，
有發ofter的二間，
總算結束了這段水深火熱的就活。

工作心得

因為公司同事不是有留外經驗，
就是末來有出國打算，
所以相對來比起傳統的日企來說，
社風感覺比較開放，
在參考價值上會比較低。

其中同事也有日本以外國家來的，
所以有時候只能用英文溝通。
個人的英文能力在進這間公司之前，
多數都只用在讀的方面，
除了偶而玩遊戲和查工作上需要的資料、
其它幾乎不怎麼接觸英文。
以量化的能力來說，
就只有檢定在國二做過全民英檢，
還是初級只過初試的慘烈程度。

但在這半年下來強迫需要用英文的結果，
聽力真的好很多，
可以實際感受到自己有明顯的進步，
自己覺得很差的口說也被澳洲同事說進步很多了，
有時候可能還比日文來的好。
出乎原先來日本的預期。

這方面覺得環境對語言的進步幫助不少，
真心推薦有機會的話，
請務必讓自己沉浸在類似的情況之中。

而工作上最常用的就是slack、quip、asana(最近在嘗試用Jira中)
基本上也是跑scrum的方式
截至寫文這個時間點是己經在跑第八個衝刺了
每個人對於工作的裁量權其實不小，
對於產品的改善與需求都能夠提出自己的意見並被採納。

唯一目前比較有微詞的是工作設備吧……
現在是帶自己2012年mid的mbp，
在building的時候真的是會等到煩躁XD
沒調校的情況下跑一次要5分多鐘，
調過還是要跑快2分，
之前老闆說好要提供到現在還是拖著，
應該哪天受不了就會自己直接買了吧。
沒想到在國外還是會有跟台灣一樣的問題……

目前就勞簽正在申請中，
在wh簽己經過期的情況下，
真不知道如果沒過的話會變的多刺激XD

下一篇的番外篇，
應該就是這系列的最後一篇了，
內容會寫工作以外在日本生活的感想與建議，
如果有什麼問題的話，
也歡迎留言詢問。

WALT株式会社

做比特幣支付服務的公司。
面試是跟創辦人與技術負責人，
原本以為二個人都還是大學生，
後來二面才知道技術負責人現在是國三？！
這時候就在想日本有很多很早創業的人是真的，
在台灣聽到說要做新創的人，
都己經是有一定年紀、資歷的了。

這間公司就服務面來說，
我並沒有覺得很突出的部份，
但人數少（三人）加上很有機會可以主導產品走向，
所以還是跟這間公司總共面談了四次。

但最後因為對方公司預計11月底左右要尋求A輪的投資，
目前seed輪似乎只有日幣千萬等級，
所以資金上還不是很充裕……
在沒辦法立刻簽雇傭契約的狀況下，
我不想要冒著可能拿不到就勞簽證的風險，
就回絕對方了。

結果：面試近一個月，2017/01上工，400~500/年，回絕。

Origami Inc.

做行動電子支付的服務。
因為在台灣己經很多家有在弄，
自己也沒什麼使用上的需求，
所以反而對這方面興趣不是很大，
就只是抱持著看看的心情去聊。

對談的是人事加上創始核心團隊成員，
聊到一半也可以感覺對方對自己沒什麼興趣，
並沒有特別詢問個人相關的問題。

公司本身風氣還蠻活潑，
比較接近外商的感覺，
辦公空間以外也有人在free space載耳機工作，
可以看的出來還蠻自由的。

然後辦公室裡也有使用電子支付的小商店，
提供給社員購買一些簡單的輕食。
這點我認為還蠻不錯的。
我一直認為如果服務的本身連公司自己人都沒在用的話，
很難說服我它會是一個多好的服務。

另外也有看到他們在東京實際推廣的店家數，
（用一塊東京地圖插上大頭針）
也會自己手做一些白板工具是蠻不錯的，
上面看的出來有在實際跑一些敏捷開發的流程，
但老實說服務面並沒有特別覺得突出的部份……

就結論是一個有點潮的外商公司，
如果對行動支付有興趣的話也許是個不錯的選擇。

結果：當天回來就收到謝謝再連絡。

株式会社スペースマーケット

做空間租借服務，
例如說想在電影院辦生日派對、在飛機庫辦社員大會等等，
跟Airbnb的類型其實蠻像的（實際上他們也有承接民宿事業）。

面談的是二位工程師，
當時會議室不夠而被請到大樓會面區（？！）聊，
當下有點在murmur。

目前實際上案件最多的類型是料理教室的租借，
其實有這樣的需求還蠻有趣的，
原先預想比較多會是想要籌辦特別活動的人才會使用，
不知道在台灣是否也有類似的服務存在。

另外有詢問對方怎麼處理雙方私下交易的問題，
但面對的是工程師加上自己沒有表達的很好，
只回答我透過平台交易比較有保障而已。
我自己在想如果只是這樣的話，
只要用任何第三方交易機制都可以避免風險了吧？
但我想這些服務通常也沒有那個資源有辦法去介入糾紛，
可惜沒有得到更好的答案就是了。

面談到後面有問到目前並沒有往國外發展的計劃，
就知道不會下文了，
但這間算是個人覺得蠻有趣的服務。

結果：二天後感謝信。

株式会社bitFlyer

比特幣交易所，
做線上比特幣買賣的服務。
認真來說這間其實不太像是新創公司…
目前都是用燒資金的方式在搶市佔率，
提供交易免手續費的條件在吸引使用者，
不得不說這間公司背後的金主很強大。

當初是在Wantedly的活動上，
被會說中文的人事聊說可以實際來辦公室逛逛聊聊，
後來還送了等值100日元的比特幣，
想說自己也看過steam上可以使用比特幣來儲值，
也許會用到就來瞭解看看。

結果實際面談的時候是負責iOS的工程師而不是當時的人事，
就在想自己當時應該是讀錯空氣了;p
面談時有詢問一些關於金流安全性的問題，
（畢竟比特幣比起本身安全性來說，管理的問題來得比較多）
只得到用離線隔離的方式在處理這一塊而已，
似乎Android端在服務本身內目前佔的重要性還不高，
實務上應該碰不到資安這一塊。

另外也詢問之後要怎麼轉虧為盈與預想的商業模式，
單純只回說要回復收手續費而己，
但這不可避免的會造成使用者的流失吧？
也沒有得到其它更好的回答……
沒聽到更多的內容我覺得有點可惜。

結果：不意外無聲卡

Sansan株式会社

名片管理服務，
算是日本特有文化才會出現的類型，
從名字就可以知道這間公司重視的是什麼:D

註：Sansan=さんさん，
さん是日文對於他人的敬稱，
相對於中文的「先生」、「小姐」的含義。

這間也不算新創，
電視上也蠻常出現他們的廣告。
會投是因為之前有做過需要影像辨識的App，
影像處理跟OCR不是一塊簡單的技術。
想知道他們是怎麼處理名片辨識的部份。

對談的是人事，
一開始就會用投影片很詳細的介紹公司的服務、組織、精神等等，
所有想要問的問題，基本上都是被講完了;P

個人最想知道的部份，
自動影像辨識不高的問題要如何解決，
其實很笨也很聰明的答案就是人工輸入XD。

目前自動辨識可以完成的部份大概佔到60%左右，
剩下的就是公司內部有專門的部門，
配合自動化流程將回報不足的部份人工補完。

還蠻意外原來做到這個規模了，
仍然可以用這樣的方式維持商業模式，
換做其它地方很有可能在一開始，
就用辨識率太低否決掉這種可能性了，
這點自己真的是學習到了。

至於面談的部份，
我很誠實的說了這次自己來日本的目標，
所以自然就是得到感謝信了。

結果：一天後感謝信，還註明很抱歉不能跟你說不採用的原因;P

Wantedly, Inc.

都用它去這麼多間了，
怎麼能不去實際朝聖一下做出這個服務的公司？;p

對談的是人事兼公關，
辦公區以外算是蠻有質感的空間。
組織的內部非常的扁平化，
幾乎都是工程師在主導。
很多開發、管理工具都是實際使用上有需求，
就自己動手弄出來，
像是圖形化數據管理介面，
連受付機也是自社開發的iOS App。
喜歡自已打造服務的話，
個人認為是一間很棒的公司。

不過實際工作氣氛其實還蠻沉靜的，
（面試官是說大家早上還沒進入工作狀況 ）
原以為會比較活潑，
卻沒什麼人在說話算是蠻意外的一點。

最近正在開發Sync的服務，
個人理解是像是Slack一樣連絡用的工具，
但相較起來它不需要是一個明確的組織內部溝通，
做一些社外、個人的連繫會比較方便。
而目前開發資源不夠所以需要找人。
這服務剛出來時有試用過一下，
但教學結束之後就發現，
自己在上面還沒有認識的人可以聊呀XD。

另外有詢問面試官自身在工作上覺得最辛苦的事，
回答之前曾經有挑戰過需要從零開始學習的業務，
雖然壓力算是蠻大的，
但如果個人想要跨入其它專業的話，
是可以提供這種環境的。

結果：當時己有其它內定，就沒談下一階段的選考。

最近終於是拿到想去公司的內定通知書了，
下星期二就準備上工，
趁現在還有時間就快把之前累積下來的寫一寫，
之後應該還會有二篇（面試、番外）。

接下來就只剩就勞簽證的申請了，
果然要開始進入日本的工作職場難免還是有點不安呀，
希望不會出什麼問題就好了。

如果有什麼問題的話也歡迎留言詢問，
以上待續。

株式会社FiNC

其實台灣也快到超老年社會的程度了，

面試官是小自己二歲工程師，

不過工作環境感覺上比較沉悶一點…
這間跟之前面試的差不多，

會播他們公司對未來願景生活的影片，

光聽日文就很吃力了還邊看影片邊聊XD

影片那種投影式鍵盤會好用嗎？

提問到最後就知道自己進不了這間公司，

溝通上交流也有感覺到障礙，
只有禁煙這點覺得很棒，
不曉得有吸煙習慣的應徵者會怎樣就是了。

株式会社Ymix

還有之前實際做過的網站。
在收集點子很多這點還蠻佩服的。
截至寫這篇文章的時候，
自己真的有想做的東西只有一個。

能想100點子，
一年挑10個有趣、可行性高的去執行，
一年後挑最有發展性的去擴展。

花許多時間培養單一的服務，

如果還有許多不確定性在的時候是一個不妨可以參考的方式。

結果：對方希望一周內給予回應，以個人專業不符主動拒絕。

dely株式会社

做食譜動畫的服務，
特點是一分鐘使用動畫讓你了解實際料理的製作方法，

這間有很多點符合自己對新創的期待，
如果當下有發ofter就會直接去的程度。

員工有三十幾個人，
面試當時工程師部份只有一個。

想說看能不能再多表現自己的企圖心XD

再加上希望現在找進來的人，
可以負責起以後帶Team的任務，

ookami株式会社

做的是運動Live的App，
希望可以結合運動直播、新聞匯整至買票服務，
一連貫的使用體驗，
個人認為有它的利基在，

目前Android的版本是外包的，
使用起來效能不是很好，
也跟現在iOS的功能有一段差距，
所以最近決定要拿回來自己推。

談起來感覺上比較開放一點，

能說英文的話比較加分。

二面是跟技術負責人
主要處理Server端跟iOS部份，
有點後悔這次自己沒有帶電腦跟Android裝置，
有些東西比起解釋，
還不如直接拿現品來看還比較清楚明瞭。

跟startup一些方便使用的CI跟工具，

要求改用Line來連絡。
但之後就這樣等了一個星期左右，
心理就有譜不會上了XD

最後收到創辦人回應他雖然個人有意願，
但以公司整體來說沒有辦法讓我加入，
(感覺上是場面話，日本這方面很難掌握。)
希望可以再保持連絡。

專業以外日文、英文其中之一要溝通無礙，

另外如果能上的話，
通常第一次聊就差不多定生死了，
當然跟HR的話例外，
談得很愉快也不代表什麼，
不得不說這方面日本給應徵者是比較好的體驗。

主動連絡而且還是社長親自寫的很意外，
但地點在港区離自已住的地方實在是有點遠，
加上考量到WH簽證和這間公司其實不小，
大概是一百多人的規模，
不然應該會試看看……
社長個人風格很強，
算是優點也是缺點吧？
只對我說Android開發有沒有信心，
就問要不要先進來試看看了。

做的內容其實很雜，
比較像SI類型的公司，
但也是有自社研發的服務。
其中比較值得一提的是 圖+吐嘈的服務，
有在看網路文章偶而會看到。

結果：試用期三個月，月薪50萬，主動放棄。

Tunnel株式会社

當天因為正在腸胃性感冒身體很不舒服。
整個身體在發冷，反應超差，
對方應該覺得很奇怪，
這人怎麼穿著外套在面試…

做的服務是室內裝飾、擺設的圖片分享服務，
面試的地方還蠻潮的，
很多東西都是使用者贈送給他們的。
以日本文化來說，
他們的死忠使用都應該不少。
自己剛搬到日本整個房間都是空的，
會想要照App裡面的佈置試看看。

隔四天後被發了感謝信，
從信裡推測原因是目前Android版才正要開始，
以需要大量溝通的前提下，
要錄用還不擅長日文的外國人，
風險還是太高了。

結果：感謝信

目前面試的心得來說，
日本的新創感覺上比台灣來的多樣性，
而且很多都是大學或是25、26歲就出來創業了，
明明台灣也有很多具有可能性的點子，
但沒那樣的投資環境，
覺得有點可惜。

然後截至目前為止都還沒有做過程式考題是蠻意外的部份，
多的只是問平常使用的framework跟工具而己，
也許日本人面試還真的蠻看重個性特質的，
不然就是我還沒到這關就被刷掉了XD

Android Studio：修改Color Theme

眼睛對於寫程式的工作來說是很重要的，
原先Android Studio的程式碼顏色顯示不喜歡，
改完設定就順手寫一下。

1.先去下載自已喜歡的Theme，

例如：ideacolorthemes

2.點選Import Settings把下載的Theme匯入

3.會問你要不要重開套用設定

4.之後就可以去設定裡微調其它的細項設定

建議可以改Theme為Darcula，
背景黑色對眼睛的負擔會比較小
